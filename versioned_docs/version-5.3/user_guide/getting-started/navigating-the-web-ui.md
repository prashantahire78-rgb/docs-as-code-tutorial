# Navigating the Web UI

## Overview

This topic explains the main areas of the StorageSphere Enterprise web UI. The navigation model groups operational tasks by dashboard, storage inventory, monitoring, alerts, reports, and administration.

Use the navigation menu to move between daily monitoring tasks and administrative workflows.

## Key Concepts

| Area | Use for |
|---|---|
| **Dashboard** | Review health, capacity, performance, alerts, and recent events |
| **Storage Inventory** | View storage systems, pools, volumes, hosts, and relationships |
| **Capacity** | Analyze capacity usage, trends, forecasts, and risk |
| **Performance** | Review IOPS, throughput, latency, and utilization metrics |
| **Alerts and Events** | Investigate active alerts and historical events |
| **Reports** | Run, schedule, export, and share reports |
| **Administration** | Manage users, roles, audit logs, collectors, and system settings |

### Common UI controls

| Control | Description |
|---|---|
| Search | Finds objects by name, identifier, vendor, site, or tag |
| Filters | Narrows lists by type, status, severity, ownership, or time range |
| Time range selector | Controls the period shown in charts and event lists |
| Column selector | Shows or hides table columns |
| Saved views | Stores frequently used filters and columns |
| Actions menu | Provides context-specific actions for selected objects |

::::tip
Use saved views for daily operations. Saved views help teams review the same systems, sites, or alert classes consistently across shifts.
::::

## Best Practices

- Start with the dashboard to identify the highest-priority operational issues.
- Use filters before exporting large inventory, alert, or event lists.
- Open object details from links instead of searching again from a different page.
- Use consistent tags for sites, environments, applications, and owners.
- Review your role if expected actions are missing from an actions menu.
